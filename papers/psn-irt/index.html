PSN-IRT: Rethinking LLM Benchmarking with Item Response Theory

Zhou et al. | HIT + CESI | AAAI 2026 Oral (arXiv:2505.15055)

📖 摘要

当前LLM评测存在两个核心问题：① 不同leaderboard之间排名不一致 ② 头部模型区分度不足。

本研究提出PSN-IRT (Pseudo-Siamese Network for Item Response Theory)，用神经网络估计IRT参数，对41,871个题目×11个benchmark进行系统分析。

关键发现：① LLM benchmarks无法在多维度同时优秀 ② 普遍存在饱和和难度上限不足 ③ 许多题目存在数据污染

🎯 方法动机

问题1：同一模型在不同benchmark上排名差异大，researcher难以判断真实能力。

问题2：头部模型（GPT-4, Claude等）在多数benchmark上接近饱和，无法区分。

解决思路：引入心理测量学中的Item Response Theory (IRT)，对题目和模型能力进行细粒度建模。

🔧 PSN-IRT 架构

双路神经网络：

  • Model Network：输入模型ID → 输出模型能力θ
  • Item Network：输入题目ID → 输出4个IRT参数

4PL (四参数逻辑) 模型：

P(X=1|θ) = c + (d-c) / (1 + e^(-a(θ-b)))

  • b (difficulty)：难度 - 正确率50%时对应的能力值
  • a (discriminability)：区分度 - 曲线陡峭程度
  • c (guessing-rate)：猜测率 - 能力极低时的正确率
  • d (feasibility)：可行性 - 能力极高时的正确率上限

📊 关键发现

1. LLM benchmarks无法多维度同时优秀
在区分度、难度范围、信度等维度上存在权衡。
2. 饱和问题严重
头部模型在多数benchmark上接近天花板，无法有效区分。
3. 数据污染普遍
许多题目被模型"记忆"，而非真正"理解"。
4. PSN-IRT筛选的题目更符合人类偏好
用高质量题目组成的子集，模型排名与人类判断更一致。

⚠️ 和你的框架的区别

PSN-IRT 建模的是：能力 vs 难度

  • θ (模型能力) × b (题目难度)
  • 关注"模型能否回答对"
  • 没有扰动维度

你的框架关注的是：扰动鲁棒性

  • 原始性能 vs 扰动后性能
  • 关注"模型对输入扰动的敏感程度"
  • 分解：扰动类型 × 模型能力 × 问题难度

潜在结合：可以结合！用PSN-IRT估计难度，用你的框架估计鲁棒性，形成二维分析。
