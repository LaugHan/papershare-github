OpenRubrics: Towards Scalable Synthetic Rubric Generation

for Reward Modeling and LLM Alignment

arXiv:2510.07743 2025-10

📝 摘要

Reward modeling是强化学习从人类反馈(RLHF)的核心，但现有奖励模型依赖标量或成对判断，无法捕捉人类偏好的多方面特性。

近期研究探索了rubrics-as-rewards (RaR)方法，使用结构化标准来捕捉响应质量的多维特性。然而，产生既可靠又可扩展的rubrics仍然是一项关键挑战。

本文提出OpenRubrics，一个大规模的(prompt, rubric)对集合，用于训练rubric生成和基于rubric的奖励模型。

核心方法：Contrastive Rubric Generation (CRG)
通过对比优选响应和拒绝响应，导出：
• Hard Rules（硬规则）：显式约束
• Principles（原则）：隐式质量

关键成果：
• 跨多个reward-modeling benchmark，Rubric-RM超越size匹配基线8.4%
• 改进迁移到policy模型，在指令遵循和生物医学benchmark上取得提升

🎯 方法动机

核心问题

现有Reward Model的局限：
• 标量评分：只输出一个数字，丢失质量的多维信息
• 成对比较：只知道A比B好，不知道为什么
• 泛化差：难以迁移到新的评估维度

Rubric的价值

Rubric（评分标准）提供了更丰富的评估结构：

  • 将质量分解为多个可解释的维度
  • 每个维度有明确的评分标准
  • 提供可操作的反馈

核心挑战

Rubric的质量和规模难以兼得：

  • 人工设计：质量高，但成本高，难以规模化
  • 自动生成：成本低，但质量参差不齐

🔬 方法设计

Contrastive Rubric Generation (CRG)

核心创新：通过对比学习生成高质量rubrics

输入：
• Prompt（任务描述）
• Preferred Response（优选响应）
• Rejected Response（拒绝响应）

过程：
→ 分析preferred vs rejected的差异
→ 提取产生差异的关键因素
→ 生成区分两者的rubric标准

输出：
• Hard Rules：preferred满足但rejected违反的显式约束
• Principles：preferred具有但rejected缺乏的隐式质量

Rubric过滤

通过偏好标签一致性过滤噪声rubrics：

  • 保留与人类偏好一致的rubrics
  • 移除与偏好冲突的rubrics
  • 确保rubric的可靠性

Rubric-RM训练

基于生成的rubrics训练奖励模型：

  • 输入：prompt + response + rubric
  • 输出：多维度评分
  • 训练目标：与人类偏好对齐

📊 实验与发现

实验设置

  • 多个reward-modeling benchmark
  • 对比：标量RM、成对RM、基于rubric的RM
  • 测试在policy模型上的迁移效果

关键结果

1. Rubric-RM效果显著提升
• 超越size匹配基线8.4%
• 在多个benchmark上一致性提升
• 证明了rubric结构的价值
2. 迁移到Policy模型成功
• 指令遵循benchmark提升
• 生物医学benchmark提升
→ Rubric不仅有助于reward modeling，还能改进下游训练
3. Hard Rules vs Principles
• Hard Rules：提供明确的约束边界
• Principles：捕捉隐式的质量信号
→ 两者结合提供更全面的评估

💡 学习应用

对Reward Modeling的启发

  • 多维优于单维：rubric提供更丰富的信号
  • 对比生成：利用preferred/rejected对生成rubric
  • 过滤保证质量：一致性过滤提高rubric可靠性

对评估研究的联系

OpenRubrics与其他论文的呼应：

IF-CRITIC：用checklist分解指令约束
DeCE：分解为Precision-Recall
General Scales：分解为18个能力维度
OpenRubrics：分解为Hard Rules + Principles

→ 共同的范式：分解才能理解

实践建议

  • 收集preferred/rejected对作为rubric生成素材
  • 使用CRG自动生成rubrics
  • 通过一致性过滤提高质量
  • 训练Rubric-RM用于下游任务

🎯 核心洞见

OpenRubrics的核心贡献：

解决了rubric生成的"规模-质量"权衡问题，通过对比学习实现自动、可靠、可扩展的rubric生成。

过去："奖励模型给响应X打了7分" → 信息有限

现在："响应X在[清晰度:5, 准确性:4, 完整性:3]维度上得分 → 更丰富的诊断信息"

这为RLHF提供了更精细的奖励信号，有助于训练更高质量的模型。
