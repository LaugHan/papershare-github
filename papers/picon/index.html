PICon: Persona Agent一致性评估框架

arXiv:2603.25620 cs.CL 2026-03-26

📝 摘要

基于大语言模型（LLM）的 persona agents 正迅速被采用，作为各个领域人类参与者的可扩展替代品。然而，目前尚无系统性方法来验证 persona agent 的回应在整个交互过程中是否避免矛盾和事实错误。审讯方法学的一个原则提供了视角：无论虚构身份多么精心设计，系统性审讯总会暴露其矛盾。

PICon 通过逻辑链多轮提问来探测 persona agents，评估三个核心维度：
• 内部一致性（无自我矛盾）
• 外部一致性（与现实世界事实对齐）
• 重测一致性（重复下的稳定性）

评估结果显示，即使被报道为"高度一致"的系统，在所有三个维度上都未能达到人类基线，揭示了在链式追问下的矛盾和回避反应。

🎯 方法动机

驱动力

LLM-based persona agents 在医疗、教育、研究等敏感领域被广泛采用，亟需系统性验证方法来确保其作为人类替代品的可靠性。

现存痛点

  • 缺乏系统化评估手段
  • 现有方法多为单轮问答
  • 无法检测长程交互中的身份漂移和矛盾
  • 无法识别回避行为

核心洞察

"无论虚构身份多么精心设计，系统性审讯总会暴露其矛盾" —— 借鉴审讯方法学原理

⚙️ 方法设计

Pipeline 流程

Step 1: 基础建立 — 建立 agent 的基本信息和背景
Step 2: 信息验证 — 通过后续问题验证已建立信息的准确性
Step 3: 矛盾探测 — 在关键时刻进行直接追问检验潜在矛盾
Step 4: 三维度评分 — 内部/外部/重测一致性综合评估

三维度评估体系

维度定义评估方式
内部一致性无自我矛盾检测回复中的逻辑矛盾
外部一致性与现实对齐验证与真实世界知识的一致性
重测一致性稳定性相同问题重复提问时的稳定性

🔬 实验结果

实验设置

  • 7组 persona agents 评估
  • 63名真实人类参与者作为基线
  • 多轮逻辑链式追问

关键发现

核心发现：即使被报道为"高度一致"的系统，在所有三个维度上都未能达到人类基线

揭示了在链式追问下的：
• 矛盾回答（Self-Contradiction）
• 回避反应（Evasive Responses）
• 身份漂移（Identity Drift）

人类 vs AI 对比

评估维度人类表现AI Agent 表现差距
内部一致性高低显著
外部一致性高中明显
重测一致性稳定不稳定显著

🎯 核心总结

一句话概括：通过系统性多轮追问框架评估 Persona Agent 的三维度一致性

Pipeline（3步）

  1. 建立 agent 基础信息 → 多轮追问探测矛盾
  2. 分析回复一致性 → 区分内部/外部/重测维度
  3. 与人类基线对比 → 揭示系统性失败模式

对 LLM 诊断方向的价值

提供了一致性检测的方法论框架，可应用于：

  • 推理过程中的自我矛盾检测
  • 多轮对话中的身份一致性监控
  • 置信度校准：一致性低 = 置信度不可靠
