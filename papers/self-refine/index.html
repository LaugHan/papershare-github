Self-Refine: Iterative Refinement with Self-Feedback

Aman Madaan et al. | CMU + AI2 + UW + Google | arXiv:2303.17651 | NeurIPS 2023
摘要（原文翻译）
像人类一样，大语言模型（LLM）并不总是在第一次尝试时就生成最佳输出。受到人类如何精炼书面文本的启发，我们提出了SELF-REFINE，一种通过迭代反馈和修正来改进LLM初始输出的方法。核心思想是使用LLM生成初始输出；然后，同一个LLM为其输出提供反馈并用它来迭代地自我修正。SELF-REFINE不需要任何监督训练数据、额外训练或强化学习，而是使用单个LLM作为生成器、修正器和反馈提供者。我们在7个多样化任务上评估SELF-REFINE，涵盖从对话响应生成到数学推理的多个领域，使用GPT-3.5、ChatGPT和GPT-4等最先进的LLM。在所有评估任务中，使用SELF-REFINE生成的输出在人类和自动指标上都优于使用相同LLM的传统单步生成，平均任务性能提升约20%。我们的工作表明，即使是最先进的LLM如GPT-4也可以通过我们简单、独立的方法在测试时进一步改进。

Self-Improvement 迭代修正 Self-Feedback 无训练

1. 方法动机

a) 驱动力

作者观察到：LLM在第一次尝试时往往不是最佳输出。特别是在多目标任务（如对话生成）或难以精确定义目标的任务（如提升代码可读性）中，初始输出可能有改进空间。

b) 现存痛点

  • 传统迭代修正：需要训练专门的修正模型，需要领域特定数据
  • 外部监督方法：需要大量训练集或昂贵的人工标注
  • Prompt方法局限：主要针对大模型有效，小模型不适用
  • 缺乏通用性：现有方法难以跨任务泛化

c) 研究假设

核心假设：LLM可以提供有用的自我反馈并据此改进自己的输出。迭代自修正（类似人类写作修改）是LLM内在能力的体现，无需额外训练即可实现。

2. 方法设计（重点）

a) Pipeline 流程

核心循环：
输入 → 初始生成 → 反馈 → 修正 → (反馈 → 修正) × N → 输出

Step 1: 初始生成（Initial Generation）

  • 输入：任务描述 x
  • Prompt：任务特定的 few-shot prompt $p_{gen}$
  • 输出：初始输出 $y_0 = M(p_{gen} ∥ x)$

Step 2: 反馈生成（Feedback）

  • 输入：原始任务 x + 当前输出 y_t
  • Prompt：反馈特定的 few-shot prompt $p_{fb}$
  • 输出：反馈 $fb_t = M(p_{fb} ∥ x ∥ y_t)$
  • 要求：反馈必须是可操作的（指出具体改进动作）和具体的（指出具体要改的地方）

Step 3: 修正（Refine）

  • 输入：原始任务 x + 当前输出 y_t + 反馈 fb_t + 历史记录
  • Prompt：修正特定的 few-shot prompt $p_{refine}$
  • 输出：修正后的输出 $y_{t+1} = M(p_{refine} ∥ x ∥ y_0 ∥ fb_0 ∥ ... ∥ y_t ∥ fb_t)$

Step 4: 停止条件（Stopping）

  • 达到指定迭代次数（论文使用最多4次）
  • 或者模型自己判断"无需进一步修正"

b) 关键设计要点

  • 同一个模型：生成器、反馈者、修正器都是同一个LLM
  • Few-shot Prompting：提供输入-输出-反馈-修正示例
  • 历史累积：保留所有历史反馈和输出，避免重复犯错

c) 公式表示

$y_0 = M(p_{gen} ∥ x)$
$fb_t = M(p_{fb} ∥ x ∥ y_t)$
$y_{t+1} = M(p_{refine} ∥ x ∥ y_0 ∥ fb_0 ∥ ... ∥ y_t ∥ fb_t)$

3. 与其他方法对比

a) 本质差异

Self-Refine使用同一个LLM完成"生成-反馈-修正"全流程，不需要额外训练、不需要强化学习、不需要外部验证器。

b) 实验结果对比

任务 Base (ChatGPT) +Self-Refine 提升
对话生成 40.1% 59.9% +19.8%
情感反转 11.4% 43.2% +31.8%
代码优化 23.9% 27.5% +3.6%
代码可读性 27.7% 63.1% +35.4%
首字母缩写生成 27.2% 37.2% +10.0%
约束生成 44.0% 67.0% +23.0%
数学推理 74.8% 75.0% +0.2%

c) 消融实验发现

  • 反馈质量很重要：具体可操作的反馈 > 泛泛的反馈 > 无反馈
  • 模型越大越强：GPT-4 + Self-Refine > GPT-3.5 + Self-Refine
  • 迭代有帮助：多次迭代通常优于单次修正

4. 实验表现与优势

a) 关键结论

  • 在7个任务上全面超越基线模型
  • 平均性能提升约20%
  • GPT-4通过Self-Refine可以进一步挖掘潜力
  • 对话生成任务提升最显著（GPT-4: 25.4% → 74.6%）

b) 优势场景

  • 多目标优化：需要同时满足多个条件的任务
  • 主观评价：无标准答案，需人类偏好判断
  • 开放域生成：约束生成、对话、缩写等

c) 局限性

  • 数学推理提升有限：仅+0.2%（模型难以准确判断自己是否做错）
  • 计算成本：需要多次LLM调用，延迟和成本翻倍
  • 停止条件：模型可能错误地认为"已经很好"
  • 不适用于简单任务：任务已完美解决时会浪费时间

5. 学习与应用

a) 开源情况

代码、数据和Demo全部开源：https://selfrefine.info/

b) 实现要点

  • 三个Prompt：$p_{gen}$, $p_{fb}$, $p_{refine}$需要精心设计
  • Few-shot示例：提供反馈和修正的示例很关键
  • 最大迭代：论文使用最多4次迭代
  • 温度设置：temperature=0.7

c) 迁移能力

可迁移到任何可以定义"反馈"的生成任务：

  • 代码生成与优化
  • 文本风格转换
  • 对话系统
  • 创意写作
  • 任何需要迭代改进的生成任务

6. 总结

a) 一句话概括（≤20字）

同一个LLM自我生成反馈并迭代修正输出，无需训练即可提升质量约20%。

b) 速记版Pipeline（3-5步）

  1. 先生成：用LLM生成初始答案
  2. 再反馈：让同一LLM给答案提具体、可操作的改进建议
  3. 后修正：根据反馈修改答案
  4. 循环：重复"反馈→修正"直到满意或达到次数上限
