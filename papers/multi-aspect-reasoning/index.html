What Defines Good Reasoning in LLMs?

Dissecting Reasoning Steps with Multi-Aspect Evaluation

arXiv:2510.20603 2025-10

📝 摘要

尽管LLM在各种推理任务上取得了显著进展，但"什么是好的推理"仍然缺乏清晰定义。传统评估通常将推理视为单一能力，使用最终答案的正确性来评判好坏，忽视了推理过程中的多个维度。

本文探索多方面评估(Multi-Aspect Evaluation)来解构LLM推理的多个质量维度：
• 逻辑连贯性
• 推理步骤的必要性
• 事实准确性
• 结论有效性

通过对推理步骤进行细粒度分析，本文揭示了：
• 传统评估指标（如pass@k）与推理质量维度的关系
• 不同LLM在不同推理维度上的相对优势
• 如何通过多维度诊断改进推理能力

🎯 方法动机

核心问题

传统推理评估的盲点：
• "正确答案 = 好推理"？不一定！
• 正确答案可能来自错误的推理过程
• 错误答案可能来自基本正确的推理
• CoT（思维链）让推理过程可见，但评估仍然只看结果

关键观察

推理是复杂的多维度能力：

  • 逻辑维度：推理步骤是否符合逻辑规则？
  • 事实维度：每步引用的信息是否正确？
  • 结构维度：步骤是否必要？有无冗余？
  • 结论维度：结论是否从前提有效地导出？

解决思路

将推理评估从"结果判断"变为"过程诊断"：

  • 分解推理过程为多个可评估的步骤
  • 对每步评估多个质量维度
  • 从多维度视角理解模型能力

🔬 方法设计

推理质量的多方面分解

论文定义了评估推理质量的多个维度：

  • 逻辑连贯性 (Logical Coherence)
    → 推理步骤之间是否逻辑连贯？
    → 有无逻辑跳跃或矛盾？
  • 步骤必要性 (Step Necessity)
    → 每个推理步骤是否必要？
    → 有无可以省略的步骤？
  • 事实准确性 (Factual Accuracy)
    → 推理过程中引用的知识是否正确？
    → 有无事实错误？
  • 结论有效性 (Conclusion Validity)
    → 结论是否有效地从前提导出？
    → 有无"结论先行"的问题？

评估方法

Step 1: 推理步骤提取
→ 将LLM的CoT分解为离散的推理步骤

Step 2: 多维度评分
→ 对每个步骤评估多个质量维度

Step 3: 综合分析
→ 聚合各维度得分，生成推理质量画像

Step 4: 与传统指标对比
→ 分析多维度评估与传统pass@k的关系

📊 实验与发现

关键发现

1. 推理质量 ≠ 答案正确
• 正确答案可能来自低质量的推理过程
• 错误答案可能来自基本正确的推理
→ 传统评估错过了推理过程的重要信息
2. 模型各有优劣维度
• 模型A：逻辑连贯性强，但事实准确性一般
• 模型B：事实准确性高，但逻辑跳跃多
→ 传统总分掩盖了维度的差异
3. CoT质量的多维性
• 更长的CoT ≠ 更好的推理
• 有冗余步骤的CoT质量可能较低
• 必要步骤数量是更好的质量指标

对推理研究的启发

  • 超越pass@k：使用多维度评估来诊断推理质量
  • 针对性改进：根据能力画像决定优化方向
  • CoT设计：设计更高质量的CoT模板

实践建议

评估维度检查清单：
□ 逻辑连贯性：步骤之间是否连贯？
□ 步骤必要性：每步是否必要？
□ 事实准确性：引用的知识是否正确？
□ 结论有效性：结论是否从前提导出？

🎯 核心洞见

本文的核心贡献：

揭示了推理质量的多维本质，打破了"正确答案 = 好推理"的简单假设。

过去："模型X在GSM8K上答对了 → 模型X推理好"

现在："模型X逻辑连贯性90%，但事实准确性65% → 需要提升事实知识"

这为理解和改进LLM推理提供了更精细的工具。
